Brain Tumor Classification Web App
Merged Implementation Plan
Mini Project 1.2  |  React + FastAPI + PostgreSQL + TensorFlow/Keras

Overview
A full-stack web application that classifies brain MRI scans into 4 tumor types using a CNN trained on a real labelled dataset. The app provides a severity/risk assessment in place of explicit staging, since the dataset does not contain stage labels. Predictions are stored in PostgreSQL and surfaced in a history view.

⚠ NOTE: Staging limitation: The dataset contains tumor type labels (glioma, meningioma, pituitary, no_tumor) but NO explicit stage labels. Staging is presented as a risk severity assessment (Low / Moderate / High) derived from prediction confidence. If a separate staging dataset becomes available, a second model head can be added.

3. Database (PostgreSQL)
Schema
Single table, no user authentication:

CREATE TABLE scan_records (
id               SERIAL PRIMARY KEY,
filename         VARCHAR(255) NOT NULL,
original_filename VARCHAR(255),
tumor_type       VARCHAR(50)  NOT NULL,   -- glioma | meningioma | pituitary | no_tumor
confidence       FLOAT        NOT NULL,   -- 0.0 to 1.0
severity         VARCHAR(20),             -- Low | Moderate | High
details          JSONB,                   -- full softmax scores per class
created_at       TIMESTAMP DEFAULT NOW()
);

Configuration
Connection via env var: DATABASE_URL=postgresql://user:pass@localhost:5432/brain_tumor_db
Tables created on startup (SQLAlchemy create_all) for dev; use Alembic for prod migrations
GET /history returns last N records ordered by created_at DESC

4. ML Pipeline (ml/)
Model Architecture
Base: MobileNetV2 pretrained on ImageNet (frozen initially, fine-tuned last 20 layers)
Input: 224x224 RGB images, normalized to [-1, 1] (MobileNetV2 standard)
Head: GlobalAveragePooling2D -> Dense(128, relu) -> Dropout(0.3) -> Dense(4, softmax)
Loss: categorical_crossentropy | Optimizer: Adam(lr=1e-4) | Metric: accuracy
Output: backend/models/brain_tumor_model.h5 + config.json

Severity Mapping
Since no stage labels exist, severity is derived from confidence at inference time:

Training Script (ml/train.py)
Reads from Data/Training/ and Data/Testing/ using ImageDataGenerator.flow_from_directory
Augmentation: rotation_range=20, horizontal_flip, zoom_range=0.1 (training only)
Train/val split: 80/20 from Training folder; Testing folder used as held-out test set
Saves model weights + config.json (class indices, severity thresholds) to backend/models/
Target accuracy: >85% on test set before wiring to API

Key Implementation Notes

Preprocessing must exactly match training: resize to 224x224, MobileNetV2 preprocess_input
CORS: allow http://localhost:5173 (Vite dev server)
Uploaded files saved to uploads/ with UUID filename to avoid collisions
POST /api/predict response shape: { tumor_type, confidence, severity, all_scores, image_url }

7. Running the Stack
Prerequisites
PostgreSQL installed and running on localhost:5432
Python 3.9+ with pip
Node.js 18+ with npm
Create database: CREATE DATABASE brain_tumor_db;

Step-by-Step
Step 1 — Train the model:
cd ml
pip install -r requirements.txt
python train.py
# Copies model to ../backend/models/brain_tumor_model.h5

Step 2 — Start the backend:
cd backend
pip install -r requirements.txt
set DATABASE_URL=postgresql://postgres:yourpass@localhost:5432/brain_tumor_db
uvicorn app.main:app --reload

Step 3 — Start the frontend:
cd frontend
npm install
npm run dev
# Open http://localhost:5173
